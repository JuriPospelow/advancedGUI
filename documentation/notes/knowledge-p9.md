# Software Levels & Requirements

## General Software Maturity Levels

| Level | Name | Description | Requirements applied |
|---|---|---|---|
| **L1** | **Prototype / Proof-of-Concept** | Test an idea, throwaway code | Nearly none — just functional core |
| **L2** | **Internal / Personal Tool** | Used by you or your team | Basic usability, minimal docs |
| **L3** | **Production / Client-Facing** | Delivered to a customer | Scalability, security, monitoring, tests, logging, error handling |
| **L4** | **Safety-Critical / Regulated** | Medical, aviation, automotive | Formal verification, certifications, audit trail, SIL/DAL levels |

## Industry Standards

| Standard | Levels | Domain |
|---|---|---|
| **ASIL** (ISO 26262) | A (lowest) → D (highest) | Automotive |
| **DAL** (DO-178C) | E (lowest) → A (highest) | Aviation |
| **SIL** (IEC 61508) | 1 (lowest) → 4 (highest) | Industrial safety |
| **CMMI** | 1 (initial) → 5 (optimizing) | Process maturity |
| **IEC 62304** | A (low) → C (high) | Medical software |
